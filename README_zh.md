# AIX-Flow: AI 原生原子化執行框架

[![Go Report Card](https://goreportcard.com/badge/github.com/nighteye841228/aix-flow)](https://goreportcard.com/report/github.com/nighteye841228/aix-flow)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**AIX-Flow** 不僅僅是一個下載器；它是一個**安全性優先的架構框架**，旨在解決 AI 生成軟體中的「幻覺風險」。它強制實施嚴格的**原子化治理 (Atomic Governance)** 模型，確保不可預測的 AI 邏輯不會讓您的系統陷入損壞狀態。

## 🧠 為什麼需要 AIX-Flow？ (架構性痛點)
隨著 AI 代理（如 Gemini, GPT-4）撰寫越來越多的生產程式碼，最大的威脅不再是語法錯誤，而是**不可預測的副作用**。AI 代理往往無法處理第三方 API 的極端失敗情境（例如部分 HTTP 回應、頻率限制或損壞的串流），從而導致系統發生「無聲的資料損壞」。

**AIX-Flow 的核心價值在於將「錯誤處理策略」從 AI 手中收回，交還給由人類定義的框架。**

---

## 🚀 核心架構支柱

### 1. 原子化交易治理
每一個工作單元都是一個 `Task`，且強制實施 `Undo()` 方法。
- **AI** 負責撰寫 `Execute` (執行) 邏輯。
- **資深工程師** 則手動審查並定義 **Undo (還原) 策略**，確保如果任務在*任何*時間點失敗，系統都能自動恢復到「零狀態 (Zero-State)」。

### 2. 副作用隔離
任何任務都不允許直接改動生產資料。所有的 I/O 都被重新定向到隔離的緩衝區或暫存路徑。最終的系統狀態變更必須通過經由人類驗證的 `MergeTask` 才能合併。

### 3. 智慧型「黑盒子」日誌
AI 除錯需要巨量的元數據 (Metadata)。然而，傳統的磁碟日誌會造成嚴重的 I/O 瓶頸。
- **智慧型 Ring Buffer**: 日誌平時保存在記憶體中，**僅在 Error Flag 被觸發時才寫入磁碟**。這為 AI 提供了一個完整的「案發現場快照」來進行自我除錯，同時在正常運作期間實現零效能損耗。

## 📊 量化基準測試 (架構實證)

我們在模擬的外網 (WAN) 環境下測試了 1GB 檔案的處理能力。數據證明了**原子化設計**能同時兼顧**安全性**與**極致效能**。

### A. 「智慧型日誌」的效能優勢 (100MB 任務)
*當需要記錄 AI 等級的大量元數據時，傳統日誌會讓執行速度慢下 6 倍以上。*

| 日誌策略 | 正常完成耗時 | 效能損耗 | 架構效益 |
| :--- | :--- | :--- | :--- |
| **傳統磁碟日誌** | `1.50s` | **+600%** | 頻繁的磁碟寫入造成 I/O 瓶頸。 |
| **AIX-Flow 智慧型日誌** | `0.23s` | **0%** | 記憶體等級的紀錄速度，出錯時仍保有完整追蹤資訊。 |

### B. 容錯成本比對：原子化 vs. 單體架構 (1GB 任務)
*模擬第三方 API 在下載進度 50% 時發生失敗後的復原成本。*

| 架構模式 | 順利完成耗時 | 發生 50% 失敗後耗時 | 容錯能力分析 |
| :--- | :--- | :--- | :--- |
| **傳統單體架構 (Non-Atomic)** | `12.56s` | `18.78s` | **零韌性**。必須從 0MB 開始重新下載。 |
| **AIX-Flow (並行原子化)** | `0.87s` | `1.01s` | **瞬間復原**。僅需復原並重試失敗的該 10MB 分塊。 |

## 🛠️ 人為介入與治理 (Manual Intervention)
本框架設計遵循 **Human-in-the-Loop** 開發模式。工程師必須在以下關鍵點手動介入：

1.  **錯誤邊界定義**: 在 `pkg/aixflow/atomic.go` 中，由工程師決定哪些外部 API 錯誤是可復原的。
2.  **還原邏輯驗證**: 每一項任務的 `Undo()` 邏輯必須經過人工審核，確保不會留下任何垃圾資料。
3.  **管線完整性**: 工程師必須親自驗證 SHA-256 串流實作，以防止資料完整性校驗被部分繞過。

## 📁 專案結構
- `pkg/aixflow/`: 核心原子化與預算控制引擎。
- `pkg/downloader/`: 基於 AIX-Flow 原則建立的並行下載器。
- `cmd/bench/`: 模擬 WAN 環境的基準測試套件。
- `docs/`: 循序圖與架構合約文件。
